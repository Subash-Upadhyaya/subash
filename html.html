HyperText Markup Language

HTML is what gives a webpage its structure. Every heading, paragraph, image, and link you've ever clicked on started as an HTML tag somewhere.

What is HTML?

HTML (HyperText Markup Language) tells the browser what to put on the page and what it means. Not how it looks — that's CSS — just what the content actually is. A heading, a paragraph, a link, an image.

It's a markup language, not a programming language. There's no logic, no calculations, no loops. You're annotating content so the browser understands it, and so assistive technologies like screen readers can make sense of the page.

Core Concepts

🏷
Tags & Elements
Most elements have an opening and closing tag: <p>...</p>. A few are self-closing — <img/>, <br/>, <hr/> — because they don't wrap any content.
🔗
Attributes
Extra info that lives inside the opening tag: href on links, src on images, class and id everywhere. Some are required, some optional.
🌳
The DOM
The browser parses HTML into a tree of objects — the DOM. JavaScript reads and modifies this tree to make pages dynamic. Every element becomes a node with properties you can change.
♿
Semantic HTML
Tags like <header>, <nav>, <article> carry meaning beyond just wrapping things. Screen readers, search engines, and other tools depend on this.

Page Structure

HTML <!-- boilerplate that every HTML file needs --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"/> <title>My Page</title> <link rel="stylesheet" href="style.css"/> </head> <body> <header> <nav><a href="/">Home</a></nav> </header> <main> <h1>Hello World!</h1> <p>A paragraph of content.</p> <img src="photo.jpg" alt="Description here"/> </main> </body>

Quick Reference

Tags Worth Knowing

  • <h1>–<h6> — Headings, most to least important. Only one <h1> per page ideally.
  • <p> — Paragraph. Gets display:block and margin by default.
  • <a href=""> — Hyperlink. href="" goes nowhere, # scrolls to top, external needs the full URL.
  • <ul> / <ol> — Bullet list / numbered list. Items go inside <li> tags.
  • <div> / <span> — Generic containers with no semantic meaning. div is block, span is inline.
  • <form>, <input>, <button> — User input. Always label your inputs.
  • <header>, <nav>, <main>, <footer> — Semantic layout. Prefer these over just divs.

Semantic Page Layout

A proper HTML5 page structure using semantic elements instead of just nesting divs everywhere:

<header> — logo, site-wide nav
<nav> — navigation links
<main> — primary content
<article> — self-contained piece
<aside> — sidebar
<footer> — contact, copyright, links
