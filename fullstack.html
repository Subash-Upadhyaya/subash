Web Architecture

Full Stack Development

Full stack means you can build both what users see and what happens behind the scenes by connecting the browser to a server, a database, and everything in between.

The Full Picture

A web application has three major layers. A full stack developer understands and builds all three, making them work together as one product.

🖥
Front End
Everything the user sees and interacts with: HTML structure, CSS styling, JavaScript behaviour. Built with React, Vue, or vanilla JS.
⚙️
Back End
The server that receives requests, runs business logic, and sends back data. Built with Python, PHP, Node.js or Ruby.
🗄
Database
Persistent storage for all the app's data like users, posts, reactions,comments. Options include MySQL/PostgreSQL (SQL) and MongoDB (NoSQL).
🔌
REST APIs
APIs define how the front end talks to the back end. A REST API uses HTTP methods: GET (read), POST (create), PUT (update), DELETE.
🔐
Authentication
Verifying user identity using passwords, tokens (JWT), or third-party providers (Google OAuth). Sessions keep users logged in.
🚀
Deployment
Getting your app live. Front end on Netlify or Vercel; back end on Heroku, Render, or a VPS. Environment variables keep secrets safe.

How a Request Works

1
User Action

User clicks a button or submits a form in the browser (front end).

2
HTTP Request

JavaScript sends a fetch() request to the server: GET /api/users.

3
Server Processing

PHP receives the request, validates it, then queries the database.

4
Database Query

MySQL or MongoDB finds and returns the matching data to the server.

5
HTTP Response

Server sends back a JSON response: { "status": 200, "data": [...] }.

6
DOM Update

JavaScript receives the JSON, then updates the page without a full reload.
