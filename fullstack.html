Web Architecture

Full Stack Development

Full stack means being able to build both sides, the interface people interact with, and the server doing the work behind it. Connecting those two things is where it gets interesting.

The Full Picture

A web application has three main layers. A full stack developer understands all three and can work across them though most people eventually specialise once they know enough to choose.

🖥
Front End
What users see and interact with. HTML for structure, CSS for appearance, JavaScript for behaviour. React, Vue, and Svelte are popular frameworks built on top of this.
⚙️
Back End
The server that handles requests, runs business logic, and talks to the database. Could be PHP, Node.js, Python, or many others, the browser never touches this code directly.
🗄
Database
Where everything gets stored, user accounts, posts, settings. SQL databases like MySQL use tables and rows. NoSQL like MongoDB uses documents that look like JSON objects.
🔌
REST APIs
The contract between front and back end. Uses HTTP verbs: GET (read), POST (create), PUT/PATCH (update), DELETE (remove). Returns JSON the browser can use.
🔐
Authentication
Verifying who someone is. Passwords get hashed, JWT tokens confirm sessions, OAuth lets users log in via Google/GitHub. Sessions keep users logged in across requests.
🚀
Deployment
Getting the app online. Static front end goes to Netlify or Vercel. Back end needs a server, Render, Railway, or a VPS. Environment variables keep secrets out of code.

How a Request Works

1
User Action

User clicks something, submits a form, loads a page, hits a button. The browser is ready to do something.

2
HTTP Request

JavaScript fires a fetch() call, something like GET /api/users. This leaves the browser and travels over the network.

3
Server Processing

PHP or Node.js receives the request, checks it's valid, authenticates if needed, then works out what data is required.

4
Database Query

Server queries MySQL or MongoDB. The database runs the query and sends back matching records.

5
HTTP Response

Server sends JSON back: { "status": 200, "data": [...] }. The browser's JavaScript takes it from there.

Common Stacks

Popular Tech Combinations

  • MERN, MongoDB, Express, React, Node.js. All JavaScript, front to back.
  • MEAN, Same but Angular instead of React. More structured, more opinionated.
  • LAMP, Linux, Apache, MySQL, PHP. What most shared hosting runs. What I'm learning with.
  • JAMstack, JavaScript, APIs, Markup. Mostly serverless, content pulled from APIs at build time.
